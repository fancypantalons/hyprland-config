## Wallust integration (Option B)

1. `wallust-colors-template.lua` -- a template that wallust fills in with its palette variables. When wallust runs (e.g. on wallpaper change), it generates `wallust-colors.lua` with the current palette.
2. `settings.lua` does `require("wallust-colors")` at the top to load generated colors, then sets non-color options after.
3. Seed file -- copy the current wallust values as `wallust-colors.lua` so the config works immediately even before wallust runs.

## Key translation notes

- **Gesture rewrite**: `gesture = 3, horizontal, scale:0.5, workspace` -> `hl.gesture({ fingers = 3, direction = "horizontal", action = "workspace" })` plus behavior options under `hl.config.gestures`
- **`misc:vrr = 2`** -> `hl.config.debug.vfr = true` (0.54's `vrr=2` means fullscreen-only; in 0.55 it's a boolean, and fullscreen-only behavior is `render:direct_scanout`)
- **`tap-to-click`** -> `tap_to_click` (dash -> underscore rename in touchpad)
- **`bringactivetotop`** dispatcher -> `hl.dsp.window.alter_zorder({ mode = "top" })`
- **`col.active_border`** and similar dotted names -> bracket syntax `["col.active_border"]`
- **Workspace 1-10 binds** -> `for i = 1, 10 do ... end` loops for workspace/move-to-workspace/move-silent bindings
- **`windowrulev2 =`** -> `hl.window_rule({ match = {...}, effect = true })` -- all 76 rules converted
- **Bind flag `p`** -> `dont_inhibit` in Lua flags table (LSP-compatible name)
- **Monitors**: 5 lines preserved as static `hl.monitor({...})` calls (nwg-displays dropped until upgraded)
